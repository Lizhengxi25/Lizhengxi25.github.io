2024-02-03

I'm not sure where to begin discussing the reasons and outcomes behind building this website. Beyond the spontaneous urge to document my life, it's more about naturally putting my changed mindset into practice. In any case, my blog has finally taken its initial shape.

When choosing a theme, I was immediately drawn to brewski. Its simple yet enduring style caught my eye. Its elegance is a blessing for every user. The details aren't handled carelessly; each element serves the whole, without any chaotic disruptions from local aesthetics disturbing the overall harmony. This restrained self-discipline doesn't detract from its spirit because a blog inherently signifies freedom, filled with inspiration everywhere. The deep, bounded white represents its inclusiveness, while the rich burgundy exudes its charm. This harmony-based spirit also gives it a dignified presence, making it "a pleasure to behold from afar but not to play with." As an artistic theme, there's no right or wrong, but I particularly love the simplicity of brewski’s style.

Despite its resilience, it remains a niche theme. When I first built my blog, I encountered some minor difficulties, mainly that certain feature pages wouldn't display. For example, when I started, the navigation bar only had HOME, ABOUT, and ARCHIVES, whereas the official demo included GITHUB, RSS, and IMPRESSUM. I filled in the GITHUB information in the local config file and tried adding a TEST section and page by modifying the config file and adding it to the local source. However, none of these would show up.

After searching online for ways to add pages, I found that my method was correct, but I didn’t know why they weren't displaying. Later, I stumbled upon the following code in the commit section of brewski's GitHub repository (originally in Pug; I haven't figured out how to highlight Pug yet, so I'm using JavaScript for now):

Each element in the menu is checked, and if absent, returns null.

Additionally, each time I generated a page, I noticed that HOME and ABOUT displayed their respective paths, but Impressum showed null. It turned out I hadn't filled in the Impressum information in the config file.

Therefore, I suspected that incomplete information was interrupting the menu. Sure enough, after commenting out Impressum, my GITHUB and TEST pages appeared.

Another update is that I later moved the theme's configuration into the config file within the theme folder, instead of placing a theme_config in the main config file. This makes switching themes easier and modularization a good coding practice.

2024-04-17

Here's a summary of the blog adjustments from 2024-04-15 to 2024-04-16. I'm glad that after a long day of intense effort, I discovered that Hexo versions 3 and above cannot use hexo-renderer-kramed for rendering mathematical formulas. Additionally, I realized from the syntax-error reported by hexo-renderer-pandoc that it might be a version issue. After attempting modifications, I succeeded. This progression in problem diagnosis and rigorous verification is undoubtedly a sign of improvement and a good omen. Although I haven't yet confirmed that it's a Node.js version issue, the results suggest it's likely an incompatibility between versions, which I'll verify in the future. From this experience, I also recognized the drawbacks of blindly experimenting—people can become irrational and obsessed. A good engineer should establish proper experimental and testing protocols instead of trying things mindlessly.
