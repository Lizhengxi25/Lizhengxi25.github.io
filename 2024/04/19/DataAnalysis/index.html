数据处理概述

摘要

工程思维是一名学者内化知识与外化学问的必要素养。在严谨的工程过程中，数据处理是衔接实验与理论的重要步骤。本文将以IBDP物理学科所授知识与方法为基础，总结基本的数据处理原则与方法；并从个人经历出发，以实操的角度探讨数据分析中高质量的代码框架。

一、数据处理

数学部分

有效数位与科学计数法

概念

一个数据的 有效数位 (significant figures) 是将其写成科学计数法后的数字的总位数，用来描述数据的精度。区分表示数量级的数字与有效数字是十分重要的，它有以下规则：

  • 对于数据中的0的规则：如缺少它则与原来的数大小不相等，那么它便是占位零，属于表示数量级的数字，并非有效数字，如\(200\)的有效数字为\(1\)位——\(2\)；如缺少它则原来的数大小相等，那么它便是表示精度的数字，属于有效数字，如\(0.50\)的有效数位为\(2\)位——\(5.0\)。
  • 写成用科学计数法表示的数时，系数\(a\)需要保证\(a \in [1,10)\)且\(a\)不为分数。
运算规则
  • 加法与减法运算中，结果的小数数位与运算数字中小数数位最低一项的小数数位一致。
  • 乘法、除法、指数运算，结果的有效数位与运算数字中有效数位最低一项的有效数位一致。
  • 运算结果需进行四舍五入。

误差分析

误差 (error)
误差种类
  • 系统误差
  • 随机误差
不确定性 (uncertainty)
读数不确定性计算

测量量 +- 最小分度值除以2： \(a = a_0 \pm \Delta{a}, \Delta{a} = {min / 2}\)
其中，结果的精度需要与\(\Delta{a}\)一致，一般保留\(1\)位小数。

重复实验不确定性计算
  • 平均值：\(\overline{x} = \frac{1}{n}{\sum_{i = 1}^{n}{x_i}}\)
  • 标准误差：\(\delta = \sqrt{\frac{\sum_{i = 1}^{n}{(\overline{x} - x_i)^2}}{n - 1}}\)
  • 误差：\(\Delta{x} = \frac{x_{max} - x_{min}}{2}\)
不确定性的分类
  • 绝对不确定性：\(y = \Delta{a}\)
  • 分式不确定性：\(y = \frac{\Delta{a}}{a_0}\)
  • 比例不确定性：\(y = \frac{\Delta{a}}{a_0}\times{100\%}\)
不确定性的合并
  • 加法与减法运算：\(c = a \pm b, \Delta{c} =\Delta{a} + \Delta{b}\)
  • 乘法与除法运算：\(y = \frac{ab}{c}, {\frac{\Delta{y}}{y} = \frac{\Delta{a}}{a} + \frac{\Delta{b}}{b} + \frac{\Delta{c}}{c}}\)
  • 指数运算：\(c = {a^n}, \frac{\Delta{c}}{c} = \vert{n\frac{\Delta{a}}{a}}\vert\)

实验部分

一个实验至少需要改变6次自变量。每一次改变自变量得到因变量时，都需要做3-5次重复实验，取平均值，计算不确定性。

应用部分

图像与误差棒 (error bars)

误差棒

误差棒的绘制用于描述不确定性，一般包含x轴和y轴两个方向。

拟合
  • 最差拟合：两条符合正确拟合范围的线，一条最陡，一条最平缓。
  • 最优拟合：最能描述多点分布的线

在线性化拟合之后，通过求出斜率的不确定性和交点的不确定性，可以得到拟合的不确定性。斜率与交点的不确定性都与求不确定性的方法相同，而求解过程中的极值便为两条最差拟合直线。

  • 斜率的不确定性为\(m_{best} + \Delta{m} = m_{best} + \frac{m_{max} - m_{min}}{2}\)
  • 交点的不确定性为\(b_{best} + \Delta{b} = b_{best} + \frac{b_{max} - b_{min}}{2}\)。

二、代码构建

为数据分析服务的程序首先应该确保模块的可重复性，即模块的再使用性。比如一个独立实验可能会做很多次，一次实验里头也可能会重复改变很多次自变量。那么此时如何开数组、如何传参、如何输出就都很讲究。

数据与变量

简述数据存储

我认为数据与变量的存储是构建数据分析的代码框架极其重要也是最基本的一点，毕竟程序便是由无数的数据组成，因此高效、清晰的数据传递是一个程序高效的基础。

而我想提出的程序框架的第一点便是从认知上要将程序作为分析数据的辅助工具，而非整个实验的核心模块。合理且高效的数据流程图永远是程序读取数据，加以分析，再输出。永远不要将原始数据、分析结果与程序中的变量混为一谈，他们三者应该始终保持彼此独立。因此，我建议将原始数据、分析结果分两个文件存储。不要用程序直接对原始数据进行写回操作，因为程序充满着不确定性，而这种不确定性与对实验原始数据始终保持的敬意的思想相违背。即使是调试好的程序，也不应该直接对原始数据进行写入操作，因为需求与想法是迭代的，一份原始数据的纯粹对于数据分析的迭代而言至关重要。

并且，不要把原始数据写在程序里。第一，绝大多数语言的对于数据的存储远远不如Excel或者其他工具那样直观。很容易出现数据的丢失以及错误处理。第二，大多数时候一个实验是合作完成，而且常常需要外界的帮助。那么此时原始数据的共享一定是Excel最为优质，因为不是所有人都能看懂程序，也不是所有人都需要去处理这份数据。

善用类与封装（待补充高阶CS知识）

用类来封装实验中的变量是一个很好的选择，因为它会比直接开数组更清晰。对于类的使用，我认为只要一个变量有分支变量，比如不确定性或者基本变量（这个名字是我自己取的，如边长是面积的基础变量），就可以用类来封装，比如一个自变量或者一组实验。在封装时，所有的涉及赋值或者传值的函数需要尽可能地确保能够返回一个值，而不是直接进行赋值。这样对于后续的拓展而言是非常方便的。下面，我将给出一些示例代码。

可扩展性

其实，以上的全部内容都是为可扩展性而服务的。一个研究或者实验是在动态迭代与进步的，那么为之而服务的程序也应该尽可能地承前启后，保留有意义的模块。这样一方面可以节省大量时间；另一方面对于研究而言，一个优质的程序也是一份宝贵的工程日志，能让我们在过往中看到进步与激发灵感。

其实，一个程序的可扩展性，不仅仅体现在单独的实验中，也可以作为日后的模板使用。
