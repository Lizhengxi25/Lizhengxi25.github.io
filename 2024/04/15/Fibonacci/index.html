Reinterpreting the Fibonacci Sequence

Abstract

An assignment in IBDP Computer Science about recursively calculating the Fibonacci sequence inspired me to think about computing its time complexity. One task was to implement the simplest recursion, which I will refer to as "naive recursion" in the following text, defined in the first section. In this article, I first aim to prove that the time complexity of using naive recursion to compute the ( n )-th term of the Fibonacci sequence is ( O(m) ), where ( m = S_{n - 1} ), the sum of the first ( n - 1 ) terms of the Fibonacci sequence. Secondly, from this proof, we can observe more properties of the Fibonacci sequence and attempt to understand its essence from another perspective.

1

First, let me clarify the definition of computing the ( n )-th term ( F_n ) of the Fibonacci sequence using naive recursion.

Code definition:

Let ( W(F_n) ) denote the time complexity of using naive recursion to compute the ( n )-th term of the Fibonacci sequence. Since each call to this function has a time complexity of ( O(1) ), the time complexity ( W(F_n) ) is the number of times this function is called when computing ( F_n ) using naive recursion.

First, let's decompose ( F_n ) from top to bottom. Note that in each decomposition—each layer—there must be and only be two adjacent terms in the Fibonacci sequence.

If we define the first two terms ( F_1 ) and ( F_2 ) of the Fibonacci sequence as the base, it's not difficult to find that when computing the ( n )-th term, excluding the first layer ( F_n ), the computation can be decomposed into ( n - 2 ) layers, where the ( i )-th layer is ( a_i F_i + b_i F_{i - 1} ), with ( i ).

Also, we define ( a_i ) as the coefficient of the term with the larger number of items in the ( i )-th layer, and ( b_i ) as the coefficient of the term with the smaller number of items, ( i ). Moreover, since in each layer we only decompose the term with the largest number of items, in the program implementation, we only call the naive recursive function ( a_i ) times. Therefore, the time complexity of this naive recursion is the sum of the coefficients of the term with the larger number of items in each layer, plus the initial recursive call ( O(1) ), that is, ( _{i = 1}^{n - 2} a_i + 1 ).

Since ( b_i > 0 ), we can infer that ( a_{i + 1} > b_{i + 1} ), and ( a_1 = b_1 ), with ( i ). In other words, in each layer, the coefficient of the term with the larger number of items is not less than that of the term with the smaller number of items. Next, substituting the second equation into the first, we can obtain the general formula for the transformation of the coefficient of the term with the larger number of items.

[ a_{i + 1} = a_i + b_{i - 1}, i ]

That is:

[ a_i = a_{i - 1} + b_{i - 2}, i ]

And

[ a_1 = b_1 = 1 ]

[ a_2 = 2 ]

Returning to the previously mentioned time complexity ( {i = 1}^{n - 2} a_i + 1 ), we can denote the ( 1 ) as ( a_0 ), obtaining ( {i = 1}^{n - 2} a_i + 1 = {i = 0}^{n - 2} a_i ). The sum ( {i = 0}^{n - 2} a_i ) is the sum of the first ( n - 1 ) terms of the Fibonacci sequence starting with two ( 1 )'s, i.e., ( S_{n - 1} ). Therefore, ( W(F_n) = O(S_{n - 1}) + O(S_{n - 2}) ).

3

Combining the above proof and assumptions about the time complexity, we can discover a property of the Fibonacci sequence where the first two terms are both ( 1 ):

[ S_n = S_{n - 1} + S_{n - 2} + 1, n > 3 ]

Proof is as follows:

Given that the time complexity of ( F_n ) is ( O(S_{n - 1}) ), the time complexity of ( F_{n - 1} ) is ( O(S_{n - 2}) ), and the time complexity of ( F_{n - 2} ) is ( O(S_{n - 3}) ).

According to the definition and assumptions of the time complexity of naive recursion (the time complexity of a function call is ( O(1) )), we can derive:

[ W(F_n) = W(F_{n - 1}) + W(F_{n - 2}) + 1 ]

Substituting the conclusions, we get:

[ O(S_{n - 1}) = O(S_{n - 2}) + O(S_{n - 3}) + 1 ]

That is,

[ S_n = S_{n - 1} + S_{n - 2} ]
